Types

AuthResponseModel

Description

Tokens and public user data returned after successful authentication.

Fields
Field Name Description
accessToken - String! Short-lived JWT used as the Bearer token for protected operations.
refreshToken - String! Opaque token used only to obtain a rotated token pair.
user - UserModel! Public data for the authenticated user.
Example
{
  "accessToken": "abc123",
  "refreshToken": "xyz789",
  "user": UserModel
}
Types

AuthSessionModel

Description

Metadata for an active user session. No session tokens are exposed.

Fields
Field Name Description
createdAt - DateTime! Time when the session was created.
deviceId - String! Stable client device identifier associated with the session.
deviceName - String Optional human-readable device name.
isCurrent - Boolean! Whether this is the session that made the request.
lastUsedAt - DateTime! Time when this session last authenticated a protected request.
sessionId - String! Server-generated identifier used to revoke this session.
Example
{
  "createdAt": "2007-12-03T10:15:30Z",
  "deviceId": "abc123",
  "deviceName": "xyz789",
  "isCurrent": true,
  "lastUsedAt": "2007-12-03T10:15:30Z",
  "sessionId": "xyz789"
}

  "password": "xyz789"

Types

LoginResponseModel

Description

Result of a password sign-in attempt, including the 2FA branch when required.

Fields
Field Name Description
accessToken - String Access JWT. Present only when requiresTwoFactor is false.
refreshToken - String Refresh token. Present only when requiresTwoFactor is false.
requiresTwoFactor - Boolean! True when verifyTwoFactorLogin must be called before tokens are issued.
twoFactorChallengeToken - String Temporary challenge required only when requiresTwoFactor is true.
user - UserModel Authenticated user. Present only when requiresTwoFactor is false.
Example
{
  "accessToken": "abc123",
  "refreshToken": "xyz789",
  "requiresTwoFactor": true,
  "twoFactorChallengeToken": "xyz789",
  "user": UserModel
}

TwoFactorSetupModel

Description

One-time setup material used to enroll a TOTP authenticator.

Fields
Field Name Description
manualEntryKey - String! Manual TOTP secret for authenticators that cannot scan the QR code.
otpauthUri - String! otpauth URI that can be opened by a compatible authenticator app.
qrCodeDataUrl - String! Data URL containing a QR code for the otpauth URI.
Example
{
  "manualEntryKey": "abc123",
  "otpauthUri": "xyz789",
  "qrCodeDataUrl": "abc123"
}
